# Azure SQL Enterprise Optimisation – Overview

## Scenario

NHS acute trust consolidating cancer and elective pathway workloads onto Azure SQL Database in UK South, with DR in UK West.

## Objectives

- Production-grade Azure SQL architecture suitable for an enterprise DBA.
- Performance tuning and optimisation aligned to DP-300.
- Security, compliance, HA/DR, monitoring and cost optimisation.

## Repository Structure (High Level)

- \`docs/\` – Architecture and design documents.
- \`database/\` – Schema and sample workload.
- \`optimisation/\` – Index, Query Store and statistics tuning scripts.
- \`security/\` – RBAC, Entra ID, RLS and data protection artefacts.
- \`operations/\` – Monitoring, alerts, runbooks, backup and DR playbooks.
- \`cost/\` – SKU analysis and right-sizing notes.
- \`reference/\` – Mapping to DP-300 skills and external resources.
